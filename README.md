# Contributing

This is an open source book and we welcome contributions from all.
Check the issues list for inspiration.
Reporting bugs or requesting new content, by posting an issue, is also a welcome contribution.

## Procedure

1. Fork this repository.
1. Make changes in your fork.
1. File a pull request on this repository. If your changes address an existing issue, please mention that issue by number, i.e. `#1234` in the pull request.
1. An editor will review the pull request and one of the following three things will happen:
   * The pull request is accepted, i.e. your changes are merged into this repository.
   * The editor requests changes. In this case, continue work in your repo, pushing changes to the same branch that you used for the pull request.
   * The editor makes change to your pull request and then merges it. We use this approach sparingly and will communicate intent to do so with contributors before going ahead.

## Requirements

1. Adhere to the book's notation guide.
1. Hand-drawn figures must be submitted in SVG format.
1. All other figures must be in one of the following formats:
    * SVG
    * Python code that emits either an SVG or PDF.
1. Plots must use sufficiently large axis labels and clear legends.
1. Add your name to `contributors.tex` 🚀

## Getting help

1. For general help with Git, GitHub, LaTeX, or any other tool related to this repository, consult online resources (Stack Overflow, tutorials, blogs) and ask your friends, colleagues, clasmates, etc. You can also ask for help on the [contributors' Discord channel](https://discord.com/channels/1375540266018148412/1516863253052133386).


1. Questions regarding this repository, including editorial procedures, file organization, intended content, peculiarities in how we use Git or LaTeX, etc., should be posted as an issue on this repository. Higher bandwidth discussion needed to work through issues can be found in the Discord.
